#redux 

Редакс - это менеджер глобальных состояний

State Manager - инструмент для управления изменяемым состоянием системы, во фронтенде чаще в оперативной памяти живет состояние

Управление
- хранение
- изменение
- подписка на изменение этого состояние

Редакс задает архитектурный стиль, он не зависит от жизненного цикла, связь компонентов не зависима от строения нашего интерфейса

STATE:
- Изменяемое состояние, глобальное или независимое от жизненного цикла компонента, или бизнес-логика

ACTION:
- Просто объект, который несет какую-то информацию, по которой нам нужно изменить стейт

REDUCER: 
- Принимает состояние и экшен и изменяет наше состояние посредством переданного экшена
- Должен возвращать значение и быть чистой функцией

STORE:
- Объект, который мы передаем в редюсер
- Экземпляр нашего стейт-менеджера для различных манипуляций

SELECTOR:
- Функция которая хранит логику для выборки данных из стейста что бы не дублировать состояние
- Чистая функция, логика получения значения нужной части состояния
