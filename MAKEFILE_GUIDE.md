# 🔧 РУКОВОДСТВО ПО MAKEFILE

> **Универсальная автоматизация для системы управления знаниями**

**Дата создания:** 2024-12-15

---

## 🚀 БЫСТРЫЙ СТАРТ

После добавления любого файла просто запустите:

```bash
make process
```

Это автоматически:
1. ✅ Переместит файлы из Inbox в правильные папки
2. ✅ Добавит/обновит YAML метаданные
3. ✅ Исправит теги (приведет к единой системе)
4. ✅ Проверит и исправит ссылки
5. ✅ Обновит MOCs

**Система настолько универсальна, что работает даже если вы ошиблись!** 🎯

---

## 📋 КОМАНДЫ

### Основные команды

#### `make process` — **ГЛАВНАЯ КОМАНДА**
Обрабатывает новые файлы (полный цикл):
- Перемещает файлы из Inbox
- Форматирует новые файлы
- Добавляет метаданные
- Исправляет теги
- Проверяет ссылки
- Обновляет MOCs

**Использование:**
```bash
make process
```

---

#### `make format-new` — Форматировать новые файлы
Форматирует только недавно измененные файлы.

**Использование:**
```bash
make format-new
```

---

#### `make check` — Проверить систему
Проверяет целостность системы:
- Структуру папок
- Наличие MOCs и шаблонов
- Метаданные в файлах
- Статистику

**Использование:**
```bash
make check
```

---

### Индивидуальные операции

#### `make move-from-inbox` — Переместить из Inbox
Анализирует файлы в Inbox и перемещает их в правильные папки на основе содержимого.

**Использование:**
```bash
make move-from-inbox
```

**Как это работает:**
- Анализирует содержимое файла
- Определяет категорию (Bitrix, React Native, PHP, Docker, и т.д.)
- Перемещает в соответствующую папку
- Если не может определить — оставляет в Inbox

---

#### `make add-metadata` — Добавить метаданные
Добавляет YAML frontmatter в файлы без него.

**Использование:**
```bash
make add-metadata
```

**Что делает:**
- Определяет тип файла (project, pattern, note)
- Добавляет соответствующие метаданные
- Обновляет `created` и `updated` даты

---

#### `make fix-tags` — Исправить теги
Приводит все теги к единой иерархической системе.

**Использование:**
```bash
make fix-tags
```

**Что исправляет:**
- `#react #reactnative` → `#tech/react-native`
- `#bitrix` → `#tech/bitrix`
- `#php` → `#tech/php`
- `#agilelife #life` → `#life/agile`
- И другие теги по системе

---

#### `make fix-links` — Исправить ссылки
Проверяет и исправляет внутренние ссылки.

**Использование:**
```bash
make fix-links
```

**Что исправляет:**
- Старые пути (`ROAD TO THE BUSINESS/` → новые пути)
- Битые ссылки (помечает, но не удаляет)

---

#### `make update-mocs` — Обновить MOCs
Проверяет MOCs на актуальность.

**Использование:**
```bash
make update-mocs
```

---

### Форматирование

#### `make format-file FILE=путь/к/файлу.md` — Форматировать конкретный файл
Форматирует один файл:
- Добавляет метаданные
- Исправляет теги
- Проверяет структуру

**Использование:**
```bash
make format-file FILE=02-Areas/Learning/PHP/Новый\ файл.md
```

---

#### `make format-all` — Форматировать все файлы
⚠️ **ОСТОРОЖНО!** Форматирует все файлы в системе.

**Использование:**
```bash
make format-all
```

**Когда использовать:**
- После больших изменений в системе
- При массовом обновлении формата
- Для полной синхронизации

---

#### `make help` — Помощь
Показывает список всех команд.

**Использование:**
```bash
make help
```

## 🔍 КАК РАБОТАЕТ АВТОМАТИЗАЦИЯ

### 1. Определение типа файла

Система анализирует:
- **Путь файла** — где находится
- **Содержимое** — ключевые слова (паттерн, проект, компонент, и т.д.)
- **Метаданные** — если уже есть YAML frontmatter

**Примеры:**
- Файл в `01-Projects/` → type: project
- Слово "паттерн" в тексте → type: pattern
- Файл в `99-MOCs/` → type: MOC

---

### 2. Определение категории

Система определяет категорию по:
- **Содержимому** — ключевые слова (bitrix, react, php, docker)
- **Пути** — в какой папке находится
- **Тегам** — какие теги уже есть

**Примеры:**
- Слово "bitrix" → категория: bitrix
- Слово "react native" → категория: react-native
- Файл в `02-Areas/Learning/PHP/` → категория: php

---

### 3. Исправление тегов

Система автоматически:
- Находит старые теги (`#bitrix`, `#react #reactnative`)
- Заменяет на новые (`#tech/bitrix`, `#tech/react-native`)
- Сохраняет структуру файла

**Безопасно:** не удаляет ваши теги, только исправляет формат.

---

### 4. Перемещение из Inbox

Система анализирует содержимое и определяет:
- **Проект?** → `01-Projects/`
- **Паттерн?** → `03-Resources/Patterns/`
- **Bitrix?** → `02-Areas/Learning/Bitrix/`
- **React Native?** → `02-Areas/Learning/React-Native/`
- **PHP?** → `02-Areas/Learning/PHP/`
- **Docker?** → `03-Resources/Tools/Docker/`
- **Agile Life?** → `02-Areas/Health/Agile-Life/`

Если не может определить — оставляет в Inbox.

## ⚠️ ВАЖНЫЕ ЗАМЕЧАНИЯ

### Безопасность

- ✅ Все скрипты делают резервные копии перед изменением
- ✅ Не удаляют файлы (только перемещают)
- ✅ Не изменяют содержимое заметок (только форматируют)
- ✅ Можно запускать многократно (идемпотентность)

### Ограничения

- ⚠️ `make format-all` изменяет все файлы — используйте осторожно
- ⚠️ Автоматическое перемещение из Inbox может ошибиться — проверяйте результат
- ⚠️ MOCs обновляются частично — полное обновление требует ручной проверки

### Рекомендации

1. **Всегда запускайте `make process`** после добавления файлов
2. **Проверяйте результат** перед коммитом в git
3. **Используйте `make check`** для проверки системы
4. **Не бойтесь ошибаться** — система исправит большинство ошибок

## 🆘 РЕШЕНИЕ ПРОБЛЕМ

### Проблема: Скрипт не выполняется

**Решение:**
```bash
chmod +x scripts/*.sh
```

---

### Проблема: Файл переместился не туда

**Решение:**
1. Переместите вручную в нужную папку
2. Запустите `make process` снова
3. Система исправит метаданные и теги

---

### Проблема: Теги не исправляются

**Решение:**
1. Убедитесь что теги в формате `#tag` (начинаются с #)
2. Запустите `make fix-tags` отдельно
3. Проверьте результат вручную

---

### Проблема: Метаданные не добавляются

**Решение:**
1. Убедитесь что файл имеет расширение `.md`
2. Запустите `make add-metadata` отдельно
3. Проверьте что файл не в папке Templates
