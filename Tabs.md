`import { Tabs } from 'expo-router'`

Вкладки - способ навигации между различными разделами приложения.
Expo Router предоставляет макет вкладок, который может создать панель вкладок в нижней части приложения.
Самый быстрый способ начать работу - использовать шаблон.

Expo предлагает три типа навигаторов по вкладкам:
- Вкладки JS: реализованы с помощью нижних вкладок React Navigation и предлагают знакомый API, если мы уже использовали React Navigation
- Собственные вкладки: используем собственную панель вкладок платформы и предлагаем собственный внешний вид и функции.
- Пользовательские вкладки: беззаголовочные компоненты вкладок для `expo-router/ui` создания полностью настраиваемого макета вкладок для сложных шаблонов пользовательского интерфейса.

Каталог `(tabs)` - специальное имя каталога, которое указывает Expo Router использовать `Tabs` макет.
	`(tabs)/_layout.tsx` - основной файл макета панели вкладок и каждой вкладки. В нём можно управлять внешним видом и поведением панели вкладок и каждо кнопки вкладок.

	`(tabs)/*.tsx` - содержимое вкладок

`(tabs)/index.tsx` - используется по умолчанию при загрузке приложения

Для настройки вкладок можно использовать параметры из React Navigation.

Иногда нужно, что бы маршрут существовал, но не отображался на панели вкладок. Для этого можно передать в параметры href: null
```
options={{
	href: null,
}}
```

Если нужно использовать динамический маршрут, можно просто передать в параметр name динамическое название:
```
<Tabs.Screen
	name="[user]"
	option={{
		href: '/evanbacon'
		// or
		href: {
			pathname: '/[user]',
			params: {
				user: 'evanbacon',
			}
		}
	}}
/>
```

**ВАЖНО**: невозможно создать два экрана для одного и того же динамического маршрута!

`screenOptions` - объект опций, который применяется ко всем вкладкам (табам).
Это удобно, что бы не дублировать одни и те же опции для каждого `Tab.Screen`

```
screenOptions: {
	tabBarShowLabel: false 
	// отключает подписи под иконками табов.
	// полезно, если используем только иконки
	// у Tab.Screen можно переопределить
	
	tabBarItemStyle: {}
	// стили для каждого элемента таббара
	
	tabBarStyle: {}
	// ключевой блок - задаёт внешний вид самой полосы табов
}
```
