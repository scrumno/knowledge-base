# 📜 СКРИПТЫ АВТОМАТИЗАЦИИ

> Вспомогательные скрипты для автоматической обработки заметок

---

## 📋 СПИСОК СКРИПТОВ

### `format_file.sh` — Форматирование одного файла
**Назначение:** Форматирует один файл (определяет тип, добавляет метаданные, исправляет теги)

**Использование:**
```bash
./scripts/format_file.sh путь/к/файлу.md
```

**Что делает:**
- Определяет тип файла (project, pattern, note)
- Определяет категорию (bitrix, react-native, php, etc.)
- Добавляет/обновляет YAML frontmatter
- Исправляет теги в содержимом

---

### `move_from_inbox.sh` — Перемещение из Inbox
**Назначение:** Анализирует файлы в Inbox и перемещает их в правильные папки

**Использование:**
```bash
./scripts/move_from_inbox.sh
```

**Что делает:**
- Анализирует содержимое каждого файла в Inbox
- Определяет категорию на основе ключевых слов
- Перемещает в соответствующую папку
- Если не может определить — оставляет в Inbox

**Алгоритм определения:**
1. Проект? → `01-Projects/`
2. Паттерн? → `03-Resources/Patterns/`
3. Bitrix? → `02-Areas/Learning/Bitrix/`
4. React Native? → `02-Areas/Learning/React-Native/`
5. PHP? → `02-Areas/Learning/PHP/`
6. И т.д.

---

### `format_new_files.sh` — Форматирование новых файлов
**Назначение:** Форматирует все новые .md файлы (кроме системных)

**Использование:**
```bash
./scripts/format_new_files.sh
```

**Что делает:**
- Находит все .md файлы (кроме MOCs, Templates, системных)
- Форматирует каждый файл через `format_file.sh`

---

### `add_metadata.sh` — Добавление метаданных
**Назначение:** Добавляет YAML frontmatter в файлы без него

**Использование:**
```bash
./scripts/add_metadata.sh
```

**Что делает:**
- Находит файлы без YAML frontmatter
- Определяет тип файла
- Добавляет соответствующие метаданные

---

### `fix_tags.sh` — Исправление тегов
**Назначение:** Приводит все теги к единой иерархической системе

**Использование:**
```bash
./scripts/fix_tags.sh
```

**Что исправляет:**
- `#react #reactnative` → `#tech/react-native`
- `#bitrix` → `#tech/bitrix`
- `#php` → `#tech/php`
- `#agilelife #life` → `#life/agile`
- И другие теги

---

### `fix_links.sh` — Исправление ссылок
**Назначение:** Проверяет и исправляет внутренние ссылки

**Использование:**
```bash
./scripts/fix_links.sh
```

**Что исправляет:**
- Старые пути (`ROAD TO THE BUSINESS/` → новые пути)
- Помечает битые ссылки (не удаляет)

---

### `update_mocs.sh` — Обновление MOCs
**Назначение:** Проверяет MOCs на актуальность

**Использование:**
```bash
./scripts/update_mocs.sh
```

**Что делает:**
- Проверяет консистентность MOCs
- Показывает статистику файлов
- Предупреждает о возможных проблемах

---

### `check_system.sh` — Проверка системы
**Назначение:** Проверяет целостность системы управления знаниями

**Использование:**
```bash
./scripts/check_system.sh
```

**Что проверяет:**
- Структуру папок
- Наличие всех MOCs
- Наличие всех шаблонов
- Метаданные в файлах
- Статистику системы

## 🔧 ТЕХНИЧЕСКИЕ ДЕТАЛИ

### Язык
Все скрипты написаны на **Bash** и используют стандартные Unix-утилиты:
- `grep`, `sed`, `find`, `awk`
- Работают на macOS и Linux

### Зависимости
- `bash` (версия 4+)
- `make` (для Makefile)
- Стандартные Unix-утилиты

### Обработка ошибок
Все скрипты используют:
- `set -euo pipefail` — строгий режим обработки ошибок
- Проверка существования файлов перед обработкой
- Обработка пустых результатов

### Безопасность
- ✅ Не удаляют файлы (только перемещают/форматируют)
- ✅ Не изменяют содержимое заметок (только форматируют)
- ✅ Создают временные файлы безопасно
- ✅ Можно запускать многократно (идемпотентность)
