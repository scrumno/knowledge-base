## 🎯 ФИЛОСОФИЯ СИСТЕМЫ

Система построена на гибриде **PARA** (Projects, Areas, Resources, Archives) и **Zettelkasten**:

- **PARA** — для организации по действиям и проектам
- **Zettelkasten** — для связывания знаний через внутренние ссылки
- **MOCs** (Maps of Content) — для навигации по темам

### Принципы:
1. **Каждая заметка должна быть атомарной** — один концепт, одна идея
2. **Связи важнее иерархии** — используйте `[[внутренние ссылки]]`
3. **Метаданные для фильтрации** — теги и свойства для быстрого поиска
4. **Регулярное обслуживание** — еженедельный обзор и чистка

## 📝 СИСТЕМА ИМЕНОВАНИЯ

### Формат: `YYYY-MM-DD - Краткое описание.md`

**Примеры:**
- `2024-12-15 - Pattern Module для React Native.md`
- `2024-12-10 - Встреча с командой проекта X.md`
- `Bitrix - Установка и первичная настройка.md` (для справочных материалов)

### Правила:
1. **Дата для временных заметок** (встречи, идеи, задачи)
2. **Без даты для перманентных знаний** (паттерны, концепции, справочники)
3. **Используйте заглавные буквы для важных слов** (Pattern Module, React Native)
4. **Сокращения только если общепринятые** (DDD, MVC, REST)

## 🏷️ YAML-МЕТАДАННЫЕ (PROPERTIES)

### Стандартный набор для всех заметок:

```yaml
---
type: note          # note | project | meeting | daily-review | pattern | reference
created: 2024-12-15
updated: 2024-12-15
status: active      # active | archived | draft
tags: []
related: []         # Массив [[внутренних ссылок]]
---
```

### Специализированные свойства:

#### Для проектов:
```yaml
---
type: project
status: in-progress  # planned | in-progress | completed | paused
deadline: 2024-12-31
priority: high       # low | medium | high
team: []
goals: []
---
```

#### Для паттернов:
```yaml
---
type: pattern
category: react-native  # react-native | php | ddd | architecture
antipattern: false      # true если это антипаттерн
language: typescript
difficulty: intermediate  # beginner | intermediate | advanced
related-patterns: []
---
```

#### Для встреч:
```yaml
---
type: meeting
date: 2024-12-15
participants: []
project: []
action-items: []
---
```

#### Для ежедневных обзоров:
```yaml
---
type: daily-review
date: 2024-12-15
energy-level: medium  # low | medium | high
focus-areas: []
completed: []
blockers: []
---
```

3. **Избегайте слишком специфичных тегов** (лучше использовать свойства)

## 📅 ЕЖЕДНЕВНЫЕ ПРАКТИКИ

### Утро (5 минут)
1. Открыть `99-MOCs/Home.md` — главную навигацию
2. Проверить `00-Inbox/` — обработать входящие заметки
3. Просмотреть `01-Projects/` — актуальные задачи на день

### День (по необходимости)
- Делать заметки сразу в `00-Inbox/` (не тратить время на организацию)
- Создавать внутренние ссылки `[[заметка]]` на лету
- Использовать теги для быстрой категоризации

### Вечер (10 минут)
1. **Обработка Inbox** — переместить заметки в нужные папки
2. **Обновить MOCs** — добавить новые заметки в карты содержания
3. **Создать связи** — связать новую заметку с существующими через `[[]]`
4. **Ежедневный обзор** (опционально) — заполнить `Daily-Review-Template`

### Еженедельно (30 минут)
1. **Обзор проектов** — обновить статусы в `01-Projects/`
2. **Архивирование** — переместить завершенные проекты в `04-Archive/`
3. **Очистка тегов** — удалить неиспользуемые теги
4. **Обновление MOCs** — синхронизировать карты с актуальным состоянием

### Ежемесячно (1 час)
1. **Глубокий обзор** — просмотреть связи между заметками
2. **Создание новых MOCs** — если появились новые крупные темы
3. **Оптимизация структуры** — реорганизация если нужно

## 🔍 БЫСТРЫЙ ПОИСК (5 МИНУТ)

### Сценарий 1: "Нужна информация по паттерну React Native"
1. Открыть `99-MOCs/Patterns-MOC.md`
2. Найти раздел `React Native Patterns`
3. Перейти по ссылке на нужный паттерн

### Сценарий 2: "Что там с проектом Bitrix Exam?"
1. Открыть `99-MOCs/Projects-MOC.md` или `01-Projects/Bitrix-Exam/`
2. Просмотреть файл проекта и связанные заметки

### Сценарий 3: "Какие встречи были на этой неделе?"
1. Использовать глобальный поиск Obsidian: `tag:meeting`
2. Фильтр по дате в поиске

### Сценарий 4: "Что я изучал по JavaScript?"
1. Открыть `99-MOCs/Learning-MOC.md`
2. Перейти в раздел `JavaScript` или `02-Areas/Learning/JavaScript/`

### Сценарий 5: "Какая-то общая тема"
1. Использовать глобальный поиск: `Ctrl+Shift+F` (или `Cmd+Shift+F`)
2. Искать по ключевым словам
3. Использовать граф связей (`Ctrl+G`) для визуализации связей

## 🚀 МИГРАЦИЯ СУЩЕСТВУЮЩИХ ЗАМЕТОК

### Пошаговый план:

1. **Создать новую структуру папок** (см. раздел "Архитектура папок")

2. **Классифицировать существующие заметки:**
   - **Проекты** → `01-Projects/`
   - **Учебные материалы** → `02-Areas/Learning/`
   - **Паттерны** → `03-Resources/Patterns/`
   - **Справочники** → `03-Resources/Cheat-Sheets/`
   - **Docker/инструменты** → `03-Resources/Tools/`
   - **Agile Life** → `02-Areas/Health/` или `02-Areas/Personal/`
   - **Внешние ссылки** → `03-Resources/External-Links/`

3. **Добавить метаданные** в каждую заметку (можно автоматизировать через скрипт)

4. **Создать MOCs** для основных тем

5. **Восстановить связи** — добавить `[[ссылки]]` между связанными заметками

## 📚 РЕКОМЕНДАЦИИ

### DO (Делать):
✅ Создавать атомарные заметки (одна идея = одна заметка)
✅ Использовать внутренние ссылки `[[]]` активно
✅ Обрабатывать Inbox ежедневно
✅ Регулярно обновлять MOCs
✅ Использовать свойства (properties) для метаданных

### DON'T (Не делать):
❌ Создавать заметки без обработки в Inbox более 24 часов
❌ Плодить теги без необходимости (лучше использовать свойства)
❌ Дублировать информацию (лучше ссылаться)
❌ Создавать слишком глубокую иерархию папок (максимум 3-4 уровня)

## 🆘 ПОДДЕРЖКА СИСТЕМЫ

Если система начинает "засоряться":
1. Проверьте `00-Inbox/` — возможно там скопилось много необработанных заметок
2. Используйте глобальный поиск для поиска дубликатов
3. Регулярно архивируйте старые проекты
4. Периодически просматривайте граф связей — там видно "острова" несвязанных заметок
